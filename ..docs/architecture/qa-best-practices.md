# QA Best Practices & Rule Catalog

This chapter catalogs the Best Practice (`BP-*`) rule IDs defined in `SDC.Schema.QA`, and the
numbered example set generated by `SDC.Schema.QA.ExampleGenerator`.

## Rule ID format

Rule IDs follow the pattern `BP-<category>-<number>`, where **BP** stands for **Best Practice**.

| Category code | Meaning |
|---|---|
| ADH | Ad-hoc attributes |
| GEN | General/generic tree integrity |
| MUT | Mutation |
| SER | Serialization |
| VAL | Validation |

## Current rule catalog

| Rule ID | Description |
|---|---|
| `BP-ADH-001` | An `<Extension>` element should not be left with zero ad-hoc attributes and zero ad-hoc elements. |
| `BP-GEN-001` | Every non-root node in the tree must have a `ParentNode` that is itself a member of the same tree. |
| `BP-MUT-001` | Every IdentifiedExtensionType (IET) node's ID must be unique within its tree. |
| `BP-SER-001` | Serialized JSON output must not contain the internal `TreeRwLock` (a `ReaderWriterLockSlim` object). |
| `BP-VAL-001` | A node must not be left with unresolved `RejectedValues` entries from the soft-reject validation contract. |
| `BP-VAL-002` | Any coherence-rule finding emitted by `ValidateTree()` should also surface in the QA report. |

This table must be kept in sync with `SDC.Schema.QA/Rules/**/*.cs` — see the docs-sync skill.

## Numbered example set

`SDC.Schema.QA.ExampleGenerator/Program.cs` generates a fixed set of numbered example XML files
used to demonstrate/validate QA rule behavior:

| Example file | Purpose |
|---|---|
| `01-construction-basic-form.xml` | A basic constructed form, no defects. |
| `02-mutation-before-move.xml` / `02-mutation-after-move.xml` | Before/after snapshots demonstrating a tree mutation (node move). |
| `03-adhoc-attributes-mixed-namespaces.xml` | Ad hoc attributes spanning mixed XML namespaces (XML-only; JSON/BSON support for this case is a known gap). |

Code comments referencing "guide/03" or similar mean "example 03,
`03-adhoc-attributes-mixed-namespaces.xml`" above — such shorthand should be replaced with the
full file name wherever it is written, per [../conventions.md](../conventions.md).
